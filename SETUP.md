# 初回セットアップ手順書

出荷調整品LINEアラートを新しい薬局で使えるようにするまでの手順です。
所要時間は約15〜20分です。

---

## 事前に必要なもの

- パソコン（Windows / Mac どちらでもOK）
- Googleアカウント（スプレッドシート用）
- LINEアカウント（LINE公式アカウント管理用）
- 通知を届けたいLINEグループ

---

## ステップ1：Node.js のインストール

1. https://nodejs.org/ を開く
2. **LTS（推奨版）** をダウンロードしてインストール
3. ターミナル（Mac）またはコマンドプロンプト（Windows）で確認：

```
node -v
```

バージョン番号が表示されればOK。

---

## ステップ2：プロジェクトの準備

1. このフォルダ（`drug-shortage-line-alert`）をパソコンに置く
2. ターミナルでこのフォルダに移動：

```
cd drug-shortage-line-alert
```

3. 依存パッケージをインストール：

```
npm install
```

---

## ステップ3：Google Cloud の設定

Google スプレッドシートにアクセスするための設定です。

1. https://console.cloud.google.com/ にアクセス
2. 新しいプロジェクトを作成（名前は何でもOK）
3. **APIとサービス → ライブラリ** で「Google Sheets API」を検索して有効化
4. **APIとサービス → 認証情報 → 認証情報を作成 → OAuthクライアントID**
   - アプリの種類：**デスクトップアプリ**
   - 名前：任意（例：出荷調整アラート）
5. 作成後、**クライアントID** と **クライアントシークレット** をメモ
6. **OAuth同意画面** を設定
   - ユーザーの種類：**外部**
   - アプリ名：任意
   - サポートメール：自分のGmail
   - テストユーザーに自分のGmailを追加

---

## ステップ4：環境変数の設定

`.env.example` をコピーして `.env` を作成し、以下を埋める：

```
GOOGLE_CLIENT_ID=（ステップ3でメモしたクライアントID）
GOOGLE_CLIENT_SECRET=（ステップ3でメモしたクライアントシークレット）
GOOGLE_TOKENS_PATH=./credentials/tokens.json
SPREADSHEET_ID=（ステップ5で取得）
LINE_CHANNEL_ACCESS_TOKEN=（ステップ6で取得）
LINE_CHANNEL_SECRET=（ステップ6で取得）
LINE_GROUP_ID=（ステップ7で取得）
```

---

## ステップ5：Google 認証 & スプレッドシート初期化

1. Google認証を実行：

```
npm run auth
```

ブラウザが開くのでGoogleアカウントでログイン → 許可。

2. 空のGoogleスプレッドシートを新規作成し、URLからIDをコピー：

```
https://docs.google.com/spreadsheets/d/【ここがID】/edit
```

`.env` の `SPREADSHEET_ID` にこのIDを貼り付ける。

3. スプレッドシートを初期化：

```
npm run init
```

4つのシート（薬局設定・在庫品目・出荷調整DB・通知ログ）が自動で作成される。

---

## ステップ6：LINE公式アカウントの作成

1. https://developers.line.biz/ にアクセス
2. **Messaging API** のチャネルを作成
   - プロバイダー：任意の名前
   - チャネル名：例「出荷調整アラート」
3. チャネル設定画面で以下を取得：
   - **チャネルシークレット**（基本設定タブ）
   - **チャネルアクセストークン**（Messaging APIタブで「発行」）
4. `.env` に `LINE_CHANNEL_ACCESS_TOKEN` と `LINE_CHANNEL_SECRET` を記入

---

## ステップ7：LINEグループIDの取得

1. LINEグループに「出荷調整アラート」Botを友だち追加して招待
2. グループIDを取得するため、一時的にWebhookサーバーを起動：

```
node tools/webhook-server.mjs
```

3. 別のターミナルで `cloudflared` トンネルを起動：

```
npx cloudflared tunnel --url http://localhost:3002
```

表示されたURLをコピー。

4. LINE Developers のチャネル設定 → Messaging API → **Webhook URL** に貼り付けて「検証」
5. LINEグループで何かメッセージを送る → ターミナルにグループIDが表示される
6. `.env` の `LINE_GROUP_ID` に記入
7. Webhookサーバーとトンネルは停止してOK（Ctrl+C）

---

## ステップ8：LINE応答設定

1. https://manager.line.biz/ を開く
2. 作成したアカウントを選択
3. **応答設定** → **応答メッセージ** を **オフ** にする

これで、誰かがメッセージを送っても「メッセージありがとうございます」の自動返信が出なくなる。

---

## ステップ9：在庫品目の登録

スプレッドシートの **在庫品目** シートに、薬局の在庫品目を登録する。

| 品目名 | YJコード | 在庫数 | 月間出庫数 | 登録日 |
|--------|---------|--------|-----------|--------|
| アムロジピン錠5mg「サワイ」 | 2171022F1243 | 500 | 120 | （空でOK・自動入力される） |
| メトホルミン錠250mg「トーワ」 | | | | |

- **YJコード** がある品目 → コードで正確に照合される（推奨）
- **YJコード** が空の品目 → 品名で部分一致（フォールバック）
- **在庫数** / **月間出庫数** → ダッシュボードで在庫日数・緊急度を表示するために使用（空でも動作する）
- **登録日** は空のままでOK。初回の通知チェック時に自動で埋まる

---

## ステップ10：動作確認

1. 出荷調整情報を手動で収集：

```
npm run collect
```

→ 出荷調整DBシートにデータが書き込まれる。初回は変化種別が空（記録だけ）。

2. 通知チェックを手動で実行：

```
npm run notify
```

→ 初回は「変化なし」で通知されないのが正常（前日のデータがないため）。

3. 確認できたら、スケジューラーを起動：

```
npm start
```

毎日 00:00 に収集、07:00 に通知チェックが自動で動く。

---

## トラブルシューティング

| 症状 | 確認すること |
|------|------------|
| `npm run auth` でブラウザが開かない | ターミナルに表示されるURLを手動でブラウザに貼り付ける |
| `npm run init` でエラー | `.env` の `SPREADSHEET_ID` が正しいか確認 |
| LINEに通知が来ない | `.env` の `LINE_GROUP_ID` が正しいか、Botがグループに入っているか確認 |
| 「変化なし」が続く | 厚労省データの更新タイミングによるもの。データに変化があれば通知される |
| `npm start` を閉じると止まる | 現在はローカル実行のため、パソコンを起動しておく必要がある |
