# 💊 出荷調整品 LINE アラート

**調剤薬局向け — 医薬品の出荷調整変化を毎日自動で LINE に届ける Bot**

厚労省が日次更新する出荷調整 Excel（16,000 件超）を毎朝取得し、
薬局の在庫品目と照合して **「新規追加」「解除」などの変化があった品目だけ** を
LINE グループに Push 通知します。変化がない日は Bot は沈黙します。

---

## どんなものか（30 秒で分かる）

```
厚労省 Excel（16,000件以上）
    ↓  毎日 00:00 に自動取得
出荷調整 DB（前日と比較して差分を検出）
    ↓  毎日 07:00 に照合
薬局の在庫品目 × 出荷調整 DB → 変化があった品目を抽出
    ↓
変化あり → LINE グループに通知
変化なし → 何も送らない（沈黙＝問題なし）
```

### LINE に届く通知のイメージ

```
🔔 本日の出荷調整変更 2件

【新規】アムロジピン錠5mg「サワイ」
出荷調整対象となりました
②限定出荷（他社品の影響）/ 出荷量: B．出荷量減少
理由: ２．他社品の影響による需要増加
見込み: イ. 概ね1年以内に解除見込み

【解除】メトホルミン錠250mg「トーワ」
出荷調整が解除されました

詳細 → https://www.mhlw.go.jp/...
```

## 技術スタック

| 領域 | 技術 |
|------|------|
| ランタイム | Node.js（ESM） |
| データソース | 厚労省 出荷調整 Excel（xlsx で解析） |
| データ保存 | Google Sheets API（4 シート構成） |
| 通知 | LINE Messaging API（Push メッセージ） |
| スケジューラ | node-cron（00:00 収集 / 07:00 通知） |
| 管理画面 | 組み込み HTTP サーバー（CSV アップロード・在庫一覧・変化確認） |

## 照合ロジック

- **YJ コードがある品目** → コードで 1 対 1 の完全一致
- **YJ コードがない品目** → 品名の部分一致でフォールバック

## プロジェクト構成

```
├── scheduler.mjs          # スケジューラー + 管理画面サーバー
├── jobs/
│   ├── collect.mjs        # 厚労省 Excel → 出荷調整 DB に保存
│   └── notify.mjs         # 差分検出 → LINE 通知
├── scrapers/
│   └── mhlw.mjs           # 厚労省 Excel のダウンロード・解析
├── lib/
│   ├── sheets.mjs         # Google Sheets 読み書きヘルパー
│   └── line.mjs           # LINE Messaging API ヘルパー
├── tools/                 # セットアップ・デバッグ用ツール群
├── dashboard.html         # 管理画面 UI
└── mockup-*.html          # UI モックアップ
```

## セットアップ

```bash
npm install
cp .env.example .env       # 環境変数を設定
npm run auth               # Google OAuth 認証
npm run init               # スプレッドシートの 4 シートを初期化
npm start                  # スケジューラー起動（自動収集・自動通知）
```

### 必要な外部サービス

- **Google Cloud** — Sheets API の OAuth 認証
- **LINE Developers** — Messaging API チャネル
- **厚労省サイト** — 出荷調整 Excel が公開されていること（自動取得）
